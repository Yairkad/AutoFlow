## תכונות עיקריות

- **PWA מלא** – התקנה כאפליקציה על טלפון/מחשב, עבודה offline
- **נתונים מקומיים** – הכל ב-localStorage, ללא צורך בשרת
- **סנכרון ענן** – Google Sheets דרך Apps Script (דחיפה ומשיכה)
- **גיבוי ושחזור** – קובץ JSON מלא הכולל את כל הנתונים
- **ייצוא Excel** – בכל מודול, דרך SheetJS
- **הדפסה** – תצוגת הדפסה מסודרת בכל המודולים הרלוונטיים
- **נירמול תאריכים** – קלט גמיש (3/3/24, 03.03.2024 וכד') → פורמט DD/MM/YYYY
- **קוד מנהל (PIN)** – הגנה על פעולות קריטיות (מחיקה, שחזור, משיכה מענן)
- **התראות** – מלאי נמוך, חיובים קרובים, שכר ממתין לתשלום

## התקנה על הטלפון

המערכת היא PWA – ניתן להתקין אותה כאפליקציה על הטלפון ללא חנות אפליקציות.

**אנדרואיד (Chrome):**
1. פתח את האתר בכרום
2. לחץ על שלוש הנקודות (תפריט) → **"הוסף למסך הבית"**
3. אשר → האפליקציה תופיע כאייקון עצמאי

**iPhone (Safari):**
1. פתח את האתר בספארי
2. לחץ על כפתור השיתוף (ריבוע עם חץ)
3. גלול ובחר **"הוסף למסך הבית"**
4. אשר

## עבודה ללא אינטרנט

המערכת מכילה Service Worker שמאחסן את כל הקבצים במטמון (Cache) של הדפדפן.
לאחר ביקור ראשון – המערכת עובדת **ללא חיבור לאינטרנט** לחלוטין.

כל הנתונים נשמרים ב-`localStorage` של הדפדפן.

> **שים לב:** הנתונים קשורים לדפדפן ולמכשיר הספציפי. לסנכרון בין מכשירים השתמש בסנכרון ענן.

## סנכרון ענן – Google Sheets

המערכת תומכת בסנכרון דו-כיווני עם Google Sheets דרך Google Apps Script.

### הגדרת הסנכרון

1. פתח [Google Apps Script](https://script.google.com) וצור פרויקט חדש
2. העתק את קוד ה-GAS המוצג בדף **הגדרות → סנכרון ענן**
3. פרסם את הסקריפט: **Deploy → New deployment → Web App**
   - הרץ כ: `Me`
   - גישה: `Anyone`
4. העתק את ה-URL שהתקבל
5. הכנס את ה-URL בשדה **לינק סנכרון** בהגדרות

### כפתורי סנכרון
- **📤 דחוף לענן** – שולח את כל הנתונים מהאפליקציה לשיטס
- **📥 משוך מהענן** – מחליף את הנתונים המקומיים בנתונים מהשיטס (דורש קוד מנהל)
- **🧪 בדוק חיבור** – בודק שה-URL תקין

## הגדרות

בדף **הגדרות** ניתן לקבוע:
- **שם העסק** וכותרת משנה (מוצגים בכותרת כל הדפים)
- **לוגו העסק** (העלאת תמונה, נשמרת כ-Base64)
- **קוד מנהל (PIN)** להגנה על פעולות קריטיות
- **לינק סנכרון** ל-Google Sheets

## גיבוי ושחזור

### גיבוי ידני
בדף הגדרות → **💾 גיבוי JSON** – מוריד קובץ JSON עם כל הנתונים.
שם הקובץ: `גיבוי-פנצריה-DD-MM-YYYY.json`

### שחזור
בדף הגדרות → **📂 שחזר מגיבוי** – בחר קובץ JSON שגובה קודם (דורש קוד מנהל).

### ייצוא Excel
בכל מודול יש כפתור **ייצוא Excel** – מייצא את הנתונים הנוכחיים לקובץ `.xlsx`.

## קוד מנהל (PIN)

ניתן להגדיר קוד PIN (עד 8 ספרות) שיידרש לפני פעולות קריטיות:
- מחיקת נתונים
- שחזור מגיבוי
- משיכת נתונים מהענן

**הגדרה:** הגדרות → קוד מנהל → הכנס קוד → שמור
**הסרה:** הגדרות → קוד מנהל → הסר קוד

> אם לא הוגדר קוד – הפעולות מתבצעות ישירות ללא אישור.

## עדכון גרסה (Cache)

בכל עדכון משמעותי ל-`style.css` או `shared.js`:

1. עדכן את שם ה-cache ב-`sw.js`:
   ```js
   const CACHE = 'autoline-v12';
   ```

2. עדכן גם את הגרסה ב-SHELL של `sw.js`:
   ```js
   './style.css?v=1.7',
   './shared.js?v=1.7',
   ```

3. עדכן את תג ה-`<link>` וה-`<script>` בכל קבצי ה-HTML:
   ```html
   <link rel="stylesheet" href="style.css?v=1.7">
   <script src="shared.js?v=1.7"></script>
   ```

זה מבטיח שמשתמשים קיימים יקבלו את הגרסה החדשה ולא גרסה ישנה מה-cache.
